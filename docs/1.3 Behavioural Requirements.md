The following table outlines the current status of each of the behavioural requirements defined in the Requirements Document, including the extent to which our system meets each requirement, and whether each requirement has changed over the course of development.

| Req. | Description | Status | Explanation |
| :---- | :---- | :---- | :---- |
| BR1.1 | Accessible via web browser | Unchanged Fully Met | System frontend can be accessed from a web browser. |
| BR1.2 | Accessible directly via API | Unchanged Fully Met | System API can be accessed by sending direct requests to it, e.g. using Python’s requests module. |
| BR2.1 | Entering search queries | Unchanged Fully Met | Frontend provides a search function allowing users to enter a search term, which is processed by the API, which returns the results of the search. |
| BR2.2 | Search returns relevant theses | Unchanged Fully Met | Search results are returned and displayed in order of relevance to the search query |
| BR3.1 | View full thesis | Unchanged Fully Met | Where the full thesis text is available, a direct link to the PDF is displayed with each search result. |
| BR3.2 | Download full thesis PDF | Unchanged Fully Met | A separate download button was deemed unnecessary, since the direct link to the PDF is provided, from which the PDF can be downloaded. |
| BR4.1 | Displaying relevant snippets | Updated Description: ask AI how the contents of a thesis relate to or answer a specific research question. Fully Met | For each thesis where full text is available, the user can ask a thesis-specific research question, and the system returns an AI-generated explanation of the parts of the thesis that are most relevant to that question. The original 'Displaying relevant snippets' requirement is still satisfied, where instead of the exact wording of a thesis snippet being returned, it is extracted and reworded (with page numbers to reference) by Gemini before being returned.|
| BR4.2 | Page numbers displayed alongside snippet info | Fully Met | Results returned by the AI include the page numbers of the thesis that the AI used to generate the information. |
| BR5.1 | Generating AI summaries of a group of theses | Partially met | Due to context limits of the Gemini API Free Tier, we were only able to implement summarisation of a single thesis |
| BR5.2 | Summaries use citations and page numbers | Updated, Fully Met | Updated description: Summaries of each thesis use citations (when explicitly asked) and page numbers (always). Alongside the AI generated content, the page numbers used to generate the information are also displayed. Since AI summaries only operate on a single thesis, citations were not needed in this feature unless explicitly requested by a user. |
| BR5.3 | Summaries compare viewpoints across theses | Not Met | Since summaries are only implemented for single theses, rather than groups of theses, this feature was not implemented. |
| BR6.1 | Option to filter searches to be from a specific time period | Fully Met | Frontend provides input fields allowing users to input a minimum and maximum year for search results. |
| BR6.2 | Processing handwritten theses | Mostly Met | Backend utilises Optical Character Recognition to read theses whose text cannot be extracted directly, however it can sometimes produce inadequate results with handwritten scans |
| BR7.1 | FIltering based on background | Not Met | Not currently implemented. A filter by institution is a planned feature when integrating with EThOS (see Section 4.2).  |
| BR7.2 | Background information returned with each thesis | Partially Met | The author and date are returned with each search result, but no other background information is returned due to the lack of availability of such information. |
| BR7.3 | Statistics highlighting diversity of results | Not Met | This feature was not implemented. |
| BR8.1 | Thesis topic map | Not Met | The graphical topic map features were not implemented in the current system, but are planned for future development (see Section 4.2) |
| BR8.2 | Adjustable granularity of topic map | Not Met | The graphical topic map features were not implemented in the current system, but are planned for future development (see Section 4.2) |
| BR8.3 | Topic map clusters theses based on similarities | Not Met | The graphical topic map features were not implemented in the current system, but are planned for future development (see Section 4.2) |
