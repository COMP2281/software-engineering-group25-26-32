#### 3.4.2 \- Server Errors

API server errors and warnings are displayed in the terminal used to launch FastAPI. NodeJS server errors are displayed in the terminal used to launch NodeJS. If an API endpoint called as a result of frontend interaction fails, a suitable error message is also displayed on the frontend. Below is a table of common server errors and how they can be rectified (or if they can be safely ignored):

| Error message | Possible solution(s) |
| :---- | :---- |
| Warning: Database, index or IDs file not found. Please upload them through the admin panel. | Ensure the database, index and IDs files are placed as outlined in Section 3.2, or uploaded through the Setup page (see 4.1) |
| Warning: You are sending unauthenticated requests to the HF Hub. Please set a HF\_TOKEN to enable higher rate limits and faster downloads. | Can be ignored. |
| embeddings.position\_ids | UNEXPECTED  \- UNEXPECTED    :can be ignored when loading from different task/architecture; not ok if you expect identical arch. | Can be ignored |
| WARNING: CUDA not available, using CPU. Building model index will be very slow, all other features unaffected. | Can be ignored if the system does not have a CUDA-supported GPU. Otherwise, this means that PyTorch is not installed with CUDA enabled. Uninstall PyTorch and install the CUDA-enabled version for your system from [https://pytorch.org/get-started/locally/](https://pytorch.org/get-started/locally/)  |
| Search error: index out of bounds | Ensure that the index/IDs files on the system are the correct ones for the database |
| Error generating summary: Missing key inputs argument\! To use the Google AI API, provide (\`api\_key\`) arguments. | Ensure the GEMINI\_API\_KEY environment variable is set on the server |
| 400 INVALID\_ARGUMENT. {'error': {'code': 400, 'message': 'API key not valid. Please pass a valid API key.', 'status': 'INVALID\_ARGUMENT'... | Ensure the value of the GEMINI\_API\_KEY environment variable is a valid Gemini API key obtained from [https://aistudio.google.com/api-keys](https://aistudio.google.com/api-keys). |
| TESSDATA\_PREFIX environment variable not set. OCR cannot proceed. | Set the TESSDATA\_PREFIX environment variable on the server to the directory where Tessdata was installed (see Section 3.2) |
