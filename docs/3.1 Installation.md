3.1
The search model combines frotned and the backend to make a web based search application with Machine learning.the stack is made up of a python environment for the backend used for semantic search, thesis data preparation, indexing etc. and node.js on the frontend handling the web application.= that manages a user interface on a browser and connects with the backend using FAST api,it is responsible for handling logins and interaction logs aswell.

The semantic retrieval system use python and its ML libraries such as pandsas and numpy.Faiss is the vectoring library used and the model is executed with the help of pytorch and a pre trained hugging face model.the semantic model has a 100 million word parameters and is trained in under 500mb of memory.
To install the system it is as simple as having all the project files ,frontend framwork and the thesis dataset as the model preparation script trains directly from it.The script standardises thesis records as it normalises,formats and handles for duplicate entries.The script thenm extracts year values and assigns tags,Finally the thesis titles are indexed and converted to semantic vectors in the FAISS vectorspace.

The system can run on an average machine,factors such as indexing speed and responsiveness of  search is hqardware dependent. The minimum recommended specification is a dual-core 64-bit processor, 4 GB RAM, and 5 GB of free storage, while better performance is achieved with a quadcore processor , 8 GB to 16 GB RAM SSD storage, and optional GPU support for faster semantic indexing.The system uses cosine similarity to index to words.

Backend and frontend thenoperate together linked by FAST api.Node js is required to operate the frontend and the training requires python files namely prepare.py, index.py, search.py, requirements.txt, theses.csv, the Node.js application files, and the startup scripts.

Python installation creates a virtual environment and there is a requirements.txt file to easily install all dependencies by using python -m venv venv followed by pip install -r requirements.txt, These dependencies include pandas, numpy, FAISS, torch, and sentence-transformers, which support data preparation, indexing, and semantic retrieval. 
Node.js dependencies are installed separately by moving into the frontend project directory and running npm install.

After this the thesis dataset must exist in the working directory and prepared by prepare.py which cleans and structures thesis for indexing. Semantic indexing is then completed using python index.py, which generates the thesis.index and thesis_ids.npy files required for search. Installation is complete when all dependencies install correctly, the thesis dataset loads successfully, and both index files are created.
