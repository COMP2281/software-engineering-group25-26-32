# Concern 1: Users may not have a technical background, so the system must be simple to learn and use

The user demographic is researchers, where many most likely won't have much of a technical background, which is why the system was designed with a minimialist UI and simple usability such as the search bar, clearly labelled controls, and a straightfroward results page. 

The elementary UI design helps to remove the learning-curve barrier in understanding how to use the tool effectively. Core actions such as entering a query, applying filters, viewing a thesis, generating a summary, and asking a question about a thesis are all exposed through obvious interface elements rather than requiring specialist commands, knowledge, or relative experience in navigating the UI. In the words of our client, we believe that we've successfully designed the tool with simplicity: 

*"The front end and design of the tool is actually very good. I like the simplicity of it and the way each thesis has a drop-down arrow which can be expanded or collapsed while still maintaining the full search list. The functions for each individual thesis are very well set out and easy to navigate. I like the way the individual thesis page indicates the date and whether it has an abstract."*
